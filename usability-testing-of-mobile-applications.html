Usability Testing of Mobile Applications

Usability testing of mobile applications plays an important role for launching a quality mobile application. Usability testing is typically done to make sure the smooth running of application on Mobile device from usability perspective. Usability testing should occur as soon as the basic stable release for the application is ready.

Why Usability Testing of Mobile Applications

  • Browser Compatibility: Every device is having its own browser.
  • Device Compatibility: As we know, different devices render differently
  • Look and feel: Look and feel must be consistent from device to device
  • Usability is one major factor: some of the devices is touch screen and some are using Keypads.
  • Devices are classified as Low end devices and high end devices. Nokia comes under low end and other BB OS 6.0, iOS and Android devices come under high end.
  • Some requirements are device specific.
  • Device's native functionality - For example some devices, gives us the pop up and other opens a new page.
  • To verify how app responds on different networks.

Classification of usability testing of Mobile Applications

 

                                          Device Usability and Adaptability

UI Look and Feel                 Alignment and Fitment                 Multiple Device Testing

 

UI Look and Feel Testing of Mobile Applications

UI Look and Feel: Look and feel of the application is verified in this test. Different types of Look and feel test are as follows

  • GUI changes with respect to different handsets/device.
  • User input method of the devices is a major factor as there is different types of input methods available such as touch screen and Keypads.
  • For web applications, application's user interface is checked for different browsers.
  • For Native and Hybrid applications, UI is checked for different screen sizes.

Alignment and Fitment Testing of Mobile Applications

  • Application's alignment and fitment is tested with respect to different browsers on different operating systems, different screen sizes and different device types.
  • Checking access using accessories/peripherals - This verifies the application's access behavior and response by touch and by using keypad.
  • All the labels, input boxes and icons should be in place as per requirements.

Multiple Devices Testing of Mobile Applications

It is preferable to test the Mobile Application on multiple devices and multiple platforms/browsers, as numerous devices and platforms like Android, iOS, Blackberry, webOS etc are available now-a-days.

Pros:

  • It contains all the limitations and quirks present in the actual client hardware and firmware combination in the hands of target consumers

Cons:

  • Testing of mobile applications in multiple devices is expensive
  • The limited processing power and the storage of the handsets do not allow on-board diagnostic software to be loaded.
    Impossible to record the protocols.
